下面是一套**把 B2B（商用服装 AI）与 B2C（个人智能衣柜）融合在一个 App 内，并逐步走向电商变现**的完整策略。内容覆盖：定位与架构、商业模式、增长、技术方案、18 个月路线图、风险与应对、以及可借鉴案例。已充分考虑你们“AI 辅助、低成本、快迭代”的优势，因此在功能/商业设计上偏向**用户价值与可变现性优先**，而非畏惧技术复杂度。

---

## 1) 产品定位与架构

### 1.1 定位结论（组合定位）

**主定位：D「智能衣橱 + AI 购物助手」**
**副定位：B「智能穿搭助手」**
**内核能力：A「AI 服装图像工具」（B2B 变现引擎）**
**外显形态：C「轻社区（AI 时尚社区）」** 服务于导流与留存

> 解释
>
> * **高频抓手**必须来自 C 端（每天“今天穿什么” + 记录/分享），所以用 **B+D** 做高频入口与商业化主线（导流/比价/买手推荐）。
> * **B2B**是当下强现金流引擎（配额/按量付费），但天然低频；因此将其收拢在“工作台”，以工具属性承接 GMV 与口碑。
> * **轻社区**不是先做大广场，而是**基于穿搭日记与 AI 模特作品的“作品流”**；目的：为导流提供内容土壤与社交证明，提高转化与留存，而非重社交成本。

**一句话定位**（面对用户与商家）：

> “一款把**你的衣橱、你的货品和 AI** 接到一起的穿搭与生意操作系统：**今天穿什么、一键试穿、即刻上架**。”

---

### 1.2 B2B 与 B2C 的融合方式

**采用「方案 B：统一入口 + Tab 切换」，配合轻量角色感知（B+）**

* 底部固定 Tab，但**首屏排布与引导**因人而异（根据注册渠道与首日行为自动设定默认 Tab 与推荐流）。
* 任何人都能一键切换“**工作台视图**”（B2B）与“**衣柜视图**”（B2C），**同一账号、同一钱包、同一积分/配额体系**。
* 注册或从 Web 导流来的商家默认落在**工作台**，首屏出现“批量生成”“模特 12 分镜”“一键视频”；个人用户默认落在**首页/衣柜**。

> 好处：
>
> * 保留「统一生态 + 数据打通」的优势（跨旅程转化），同时不牺牲清晰度。
> * 角色感知仅用于默认排序/推荐，不做硬分叉，避免割裂。

---

### 1.3 关键用户旅程（含交叉）

**商家（B2B）**

* **D1**：注册→工作台引导→上传平铺图→生成 12 分镜/视频→首单付费
* **W1**：批量处理→一键分发到淘/拼/抖→查看素材带来的 CTR/CVR 提升
* **M1**：订阅/包量→开通**个人衣柜权益（赠送）**→开始在朋友圈晒“工作成果 + 自己的穿搭”

**个人（B2C）**

* **D1**：注册→3 步新手任务（拍衣/识别/拿推荐）→体验“AI 模特试穿”
* **W1**：每日穿搭提醒（Push）→上传/记录→分享日记（带邀请码海报）
* **M1**：被种草/比价→跳转联盟下单→对效果“有感”

**交叉桥**（核心设计）

* 在**工作台**顶部加入“**一键把店铺商品加进我自己的衣柜**”按钮；商家体验 AI 推荐与搭配，也能更好理解“搭配售卖位”的价值。
* 在**衣柜**“智能买手”中提示“**开店试试？你的衣柜里已有 30 件高流量搭配**”，一键进入“创作者/小商家工具包”（含 10 次 AI 模特体验、淘宝/拼多多上架向导、佣金任务）。
* **统一积分/配额**：拍衣、分享、带货、拉新、完成任务都返“通用点数”，可抵扣 AI 生成/会员费/商城券。

### 1.4 信息架构（IA）

**采用「方案 B：场景导向」的增强版（B+）**

```
底部 Tab：
- 首页：千人千面的智能流（穿搭卡、AI 模特作品、相似/搭配商品、商家案例）
- 创作/工作台：AI 工具集（B2B&高级 C），生成、批量、管理、任务队列
- 衣柜：拍照识别、分类、搜索、搭配、试穿、日记
- 广场：轻社区（作品流/榜单/活动/品牌页）
- 我的：账户、会员/配额、订单、分销、设置
```

> 细节：
>
> * 首页流**动态拼装**：对商家多展示“转化案例/模板/同类商家作品”；对个人多展示“今日穿搭/同城/同风格”。
> * **全局浮动按钮（+）**：拍衣/上传、生成作品、发帖。
> * **任务中心**（贯穿工作台/衣柜/广场）：所有“增长任务”集中管理。

### 1.5 核心功能 MVP 路线（按上线风险与商业价值排序）

**Phase 1（0-3 个月｜MVP）**

1. 账号&支付（手机号/微信、订阅&配额）
2. **AI 模特生成（B2B 核心）**：单图→12 分镜→视频
3. **虚拟衣柜 + 识别分类**
4. **轻量穿搭推荐**（天气/场合/通勤三卡片）
5. 任务中心 + 生成队列/历史 + 分享（含邀请海报）

> 可选：**联盟跳转的“相似/搭配商品位”**（先上淘宝/京东，抖音/拼多多次之，统一跳转打点）

**Phase 2（3-6 个月｜增长）**

1. 穿搭日记 + 轻社区（作品流、点赞/评论、榜单）
2. 商家**批量生成** + 模板中心 + 一键分发
3. 邀请分销体系（拉新返点数 + 现金佣金）
4. **电商导流（更深）**：相似/搭配商品、券信息、价格提醒
5. **角色感知**与首页个性化（冷启动特征 + 行为特征）

**Phase 3（6-12 个月｜电商）**

1. 品牌专区/联名任务
2. **二手交易内测**（基于衣柜一键发布、担保交易、物流对接）
3. **AI 购物助手**（比价、库存/券提醒、相似度搜索、AI 评测卡）
4. 数据洞察（品牌看板，**只提供汇总/匿名洞察**）

## 2) 商业模式设计

### 2.1 B2B（现有，建议增强）

* **订阅 + 按量**并存（你们已有）：

  * 基础 99 元/月/100 次；专业 299 元/月/500 次；企业定制
  * 按量：AI 模特 2 元/次；清理增强 0.3 元/次
* **加值包**：

  * **模板库/风格包**（节日、品类、场景），按包售卖（如 49-199 元/包）
  * **团队席位**（多端并发、权限、审稿流），商拍/代运营公司愿为协同付费
  * **转化看板**（外链归因+模板洞察）：提供\*\*“行业基线 vs 你的作品”\*\*对比
* **分销佣金**：30% 保留；新增\*\*“任务制”\*\*（发 3 条案例 + 1 篇教学=解锁更高分成）

### 2.2 B2C（Freemium）

* **免费**：衣柜 ≤50 件、每日 1 套推荐、日记 30 天留存
* **高级 19 元/月**：无限容量、每日 3 套、AI 试穿 10 次/月、去广告
* **尊享 49 元/月**：试穿 50 次/月、**专属“风格训练”**、品牌券包
* **点数经济**：做任务/拉新/发作品得点数，可**抵 AI 次数/会员折扣/商城券**（统一跨 B\&C）

### 2.3 电商导流（Phase 1）

* 接入**阿里妈妈·淘宝联盟**、**京东联盟**、**抖音电商（精选联盟/抖客）**、**拼多多“多多进宝/多多客”**，以 **CPS** 为主（佣金由平台类目/商品决定）。官方说明参考：淘宝联盟/阿里妈妈、京东联盟、抖音电商生态（含抖客/精选联盟）、拼多多“多多进宝”官方资料（PDF/官网） 。([Alimama][1])
* 展示**相似/搭配/口碑**卡片 + 券信息、销量/评价摘录（规范标示“广告/推广”以满足\*\*《互联网广告管理办法》可识别性\*\*要求）。([Government of China][2])

### 2.4 自营电商（Phase 2，建议先 **C2M 反向定制/预售**）

* **优先级建议**：C2M（预售先收款） > 精选买手店 > 自有品牌
* MVP 做法：

  * 以「**爆品共创**」为主题，基于衣柜数据与“愿望单/试穿热度”开款；小批量试产（100-300 件）→ 预售验证 → 放量。
  * **二段毛利模型**：预售期 60-70% 毛利（无库存），常规期回落至 40-50%。
* 运营抓手：联动**AI 模特风格包+内容挑战赛**（UGC、晒单返点数）

### 2.5 二手交易（Phase 3）

* **一键发布**：从“我的衣柜”勾选→自动带出品牌/成色/属性→AI 估价（基于同城/同品牌成交分布）
* **担保交易 + 物流对接**：对接**菜鸟裹裹/顺丰开放平台 API**（寄件取号、轨迹、电子面单等），平台收款、验收/确认后打款。([developer.alibaba.com][3])
* **收费**：成交 6-8% 手续费；可选\*\*“质检/清洗/补织”\*\*增值
* **信任**：实名认证、芝麻分/人脸（视合规与接入条件而定）、售后仲裁。

  > 说明：芝麻分/征信类对接需与蚂蚁侧合规合作，若无法接入，采用**保证金 + 平台履约险**替代。

### 2.6 收入结构（12 个月后目标区间，供 OKR）

* B2B 订阅/按量：**55-65%**
* B2C 订阅：**10-20%**
* 联盟佣金：**10-20%**（受流量结构与类目影响）
* 品牌合作/数据洞察：**5-10%**（数据需**匿名/汇总**，符合 **PIPL**）

> 合规提示：不得“出售个人数据”；若做“数据服务”，仅可提供**聚合、去标识化的趋势洞察**，遵循\*\*《个人信息保护法》\*\*。([NPC Observer][4])

## 3) 增长策略

### 3.1 冷启动（0→10,000 用户）

* **既有 Web 商家导流**：发放“App 首充礼包 + 批量模板包”；工作台首屏显著展示 Web→App 迁移收益（如本地缓存、相册拍照、离线任务）。
* **内容种草**：

  * 小红书（蒲公英/买手合作）投资源与合作达人，主题“**平价衣柜也能高定大片**”。([Xiaohongshu][5])
  * 抖音短视频：生成过程/对比前后/“100 元拍出旗舰店质感”。
* **私域裂变**：

  * **穿搭日记海报**（带邀请码）+ 新人任务
  * 商家“**案例模板挑战**”（生成 3 组对比图+上架截图，返 300 点数）

### 3.2 病毒机制

* **邀请奖励**：注册即双方各 +10 次 AI；被邀方首充，邀请人得 30% 佣金（商家带商家）；被邀方成为活跃买家（3 次下单/2 周），邀请人得**高阶模板试用**
* **UGC 激励**：周榜/月榜 + 官方推荐位；“AI 模特摄影大赛”（模板/光效包）
* **任务制裂变**：打通“分享到朋友圈/小红书/抖音”的一键海报（防封词、带短链 & 跳转统计）

### 3.3 渠道与预算（初期月 0.5-2 万）

* **有机**：小红书教程与案例、抖音演示、SEO/ASO
* **付费测试**：小红书信息流/抖音 DOU+；关注 \*\*CPA 与次留（D1/D7）\*\*差异，保留单位经济正向的组。
* **品牌/联名**：阶段性与中腰部女装品牌/买手做“风格周”，提升曝光 + 导流。

### 3.4 留存策略

* **每日穿搭 Push**（早 7-9 点，场景/天气/心情）
* **签到得点数**（连签 7 天送 1 周高级会员）
* **回流任务**（N 天不活跃 → 送 3 次 AI 试穿 + 1 张平台券）
* **商家侧留存**：批量/模板库上新订阅、行业节日活动模板、转化看板邮件周报

**北极星指标（North Star）**

* **日“有效试穿/生成”次数**（人均） + **导流点击转化贡献 GMV**（当期 + 滞后）
* 支撑指标：D1/D7/D30 留存；内容分享率；B2B 订阅留存与 ARPU；联盟 CVR

## 4) 技术实现方案

### 4.1 React Native 开发要点（结合你们现有栈）

* **框架**：React Native + Expo（EAS 分发）或裸 RN；路由建议 **Expo Router** 或 React Navigation
* **UI**：RN 组件库 + 自建 Design Token；如希望 Web/App 统一 Token，评估 **Tamagui** 做跨端样式/变量复用
* **关键页面**：

  * 首页 = 卡片流（虚拟化列表 + 分块渲染）
  * 工作台 = 任务队列/批量上传/状态轮询（WebSocket + 轮询兜底）
  * 衣柜 = 相册格子/筛选/搜索（端内 Embedding/标签）
  * 穿搭推荐 = Tinder 式卡片（左右滑/收藏/添加购物车/比价）
  * 社区 = 瀑布流（分页 + 预取缩略图）
* **性能**：

  * 图片懒加载 + 缩略图占位（blurhash）
  * 大列表用 `FlashList`/`RecyclerListView`
  * 上传前端压缩（质量/长边限制），断点续传
  * Redux Toolkit + RTK Query / Zustand；**Redux Persist** 离线缓存
* **原生能力**：相机（image-picker/vision-camera）、Push（极光/友盟或厂商通道适配）、支付（微信/支付宝）、分享（RN Share）

### 4.2 Web 与 App 代码复用

* **Monorepo（Turborepo/Nx）**

```
packages/
  shared/       # 业务可复用：api、store、utils、types、hooks、style-tokens
  ai-kits/      # 生成任务SDK、任务编排、打点、成本核算
apps/
  web/          # Next.js 14
  app/          # RN/Expo
```

* **复用率**：逻辑层 60-80%，UI 独立；API Client、数据模型、验证（zod）100% 复用
* **Feature Flag**：远程配置（ConfigCat/自建），支持灰度/AB

### 4.3 后端 API 与基础设施

* **现有 REST 复用为主**；新增：

  * **任务编排**（Redis + BullMQ）：创建/排队/进度/重试/回调
  * **推送**：事件驱动（任务完成/券提醒/价格变动）
  * **App 版本**：灰度/热更新（CodePush）+ 立即回滚能力
* **对象存储 + CDN**：阿里云 OSS/又拍等；水印、缩略图、签名 URL
* **电商联盟聚合网关**：抽象四大联盟**跳转&转链**、券查询、回流对账（阿里妈妈/京东联盟/抖音电商/拼多多“多多进宝”） 。([Alimama][1])
* **物流（二手期）**：对接菜鸟/顺丰开放 API（取号、轨迹、电子面单）。([developer.alibaba.com][3])
* **数据合规**：PIPL 数据最小化、目的限定、可撤回同意；广告标识遵循《互联网广告管理办法》“可识别性”。([NPC Observer][4])
* **App 备案**：依据工信部 2023 通知，**APP 必须完成备案并标注备案号**。([Government of China][6])

### 4.4 AI/推荐实现（MVP→进阶）

* **识别**：服装品类/颜色/纹理/季节标签（服务器端，多图加速）
* **相似/搭配**：CLIP/自研 Embedding + 近邻检索（Faiss/Elastic kNN），规则优先（季节/场合/颜色互补）→ 再加学习排序
* **试穿**：从“AI 模特 12 分镜”到“个体化身材/肤色参数化”，先**不采集真人照片**，以保护隐私，逐步引导**可选身材档位**
* **比价/券**：联盟聚合返回最低价/券，MVP 用“最低到手价 + 口碑摘要”卡片

## 5) 18 个月实施路线图（里程碑 + 核心指标）

### 0-3 个月（MVP）

* 上线：账号/支付、AI 模特生成、衣柜识别、轻推荐、任务中心、基本导流位（淘宝/京东优先）
* 指标：

  * DAU：3k-5k；首周 D1≥35%、D7≥18%
  * 商家转化：注册→首单付费 ≥15%，订阅率 ≥6%
  * 联盟 CTR：≥6%，CVR：≥1%
* 组织：核心产品/工程 1-2 人 + 设计 0.5 + 审核外包/兼职

### 3-6 个月（增长）

* 上线：穿搭日记、轻社区、批量生成、分销、深度导流（券/比价）
* 指标：

  * 用户规模：**5 万**（累计）
  * B2B MRR：≥20 万；B2C 订阅渗透 ≥3%
  * 分享率 ≥12%；次月留存 ≥28%

### 6-12 个月（电商）

* 上线：品牌专区/联名任务、二手内测、AI 购物助手（比价/提醒）
* 指标：

  * 联盟 GMV：**50-100 万/月**（佣金 5k-2.5 万区间）
  * 社区 UGC 日增：≥600 条；优质内容率 ≥15%
  * B2B 订阅续费 ≥60%

### 12-18 个月（自营/C2M & 二手放量）

* 上线：C2M 预售、精选买手品牌合作、二手全面开放（担保/物流）
* 指标：

  * 自营月销：**200 万+**（毛利 40%）
  * 二手月成交：**30-80 万**，手续费 6-8%
  * 综合营收构成：B2B 55-60%、导流/自营 25-35%、订阅 10-15%

## 6) 风险识别与应对

### 技术

* **RN 性能/端差异**：大列表卡顿、拍照兼容 —— 采用虚拟化列表、图片缩略、平台差异分支 & 充分真机测试
* **热更新风险**：CodePush/自建灰度 + 版本熔断回滚
* **AI 成本与稳定性**：任务队列限流 + 成本监控（每任务成本看板）+ 低峰批量

### 商业

* **B2B vs B2C 心智冲突**：默认 Tab + 个性化流；对商家赠送个人衣柜权益与“搭配售卖位”，用实际收益化解
* **联盟合规**：所有导流位标注“广告/推广”，遵守**广告可识别性**；素材/话术遵循《广告法》《互联网广告管理办法》。([Government of China][2])
* **自营/二手合规&供应链**：先导流/预售，再自营；小批量试产、严格验收；物流与担保流程标准化（菜鸟/顺丰 API） 。([developer.alibaba.com][3])
* **App 上线合规**：**完成 APP 备案并在显著位置标注备案号**。([Government of China][6])

### 运营

* **内容审核**：AI 预审核 + 人工复核；关键词/图像识别 + 举报通道
* **交易纠纷（二手）**：担保 + 7 天服务承诺（服装类结合成色/卫生规则制定退货细则），专属客服仲裁
* **隐私**：最小必要、同意可撤回、匿名化分析；**不得向品牌出售个人数据**（仅提供聚合洞察）。([NPC Observer][4])
