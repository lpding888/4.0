## 🎯 任务卡驱动开发模式（v4.0核心创新）⭐ 重点

### 什么是任务卡？

任务卡是老王我设计的一种标准化的开发任务格式，包含18个字段：

```json
{
  "taskId": "APP-RN-001",
  "department": "react_native_基础搭建",
  "module": "项目初始化",
  "title": "初始化React Native项目并配置基础环境",
  "phase": "Phase 1 - Week 1",
  "estimatedHours": 8,
  "priority": "P0",
  "dependencies": [],
  "needsCoordination": [],
  "description": "创建React Native项目，选择合适的版本，配置基础开发环境",
  "technicalRequirements": [...],
  "acceptanceCriteria": [...],
  "codeLocations": [...],
  "testStrategy": [...],
  "aiPromptSuggestion": "请帮我初始化一个React Native 0.72项目，要求：..." // 500-1000字详细提示词
}
```

### 任务卡的核心优势

1. **标准化**：所有任务都有统一的格式，易于管理
2. **可追溯**：每个任务都有唯一ID，依赖关系清晰
3. **可执行**：aiPromptSuggestion字段可直接复制给Claude Code
4. **可验收**：acceptanceCriteria字段明确验收标准

### 如何使用任务卡？

**方式1：直接交给Claude Code**
1. 复制任务卡的aiPromptSuggestion字段
2. 粘贴给Claude Code
3. Claude Code自动执行任务
4. 验收标准检查

**方式2：交给Product Planner规划**
1. 把用户需求交给Product Planner
2. Product Planner输出任务卡清单
3. 按照任务卡顺序执行
4. 各部门协作完成

**方式3：自己规划任务卡**
1. 参考Q14-Q19的任务卡示例
2. 自己编写任务卡
3. 交给Claude Code执行

### 任务卡的估时和排期

- **单个任务卡**：4-12小时
- **每天完成**：1-2个任务卡
- **每周完成**：8-12个任务卡
- **MVP上线**：20-30个任务卡（2-3周）
- **完整项目**：150-200个任务卡（8-12周）
