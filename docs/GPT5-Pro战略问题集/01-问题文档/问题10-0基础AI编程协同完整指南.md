     - 优势？（代码补全、广泛支持）

### 第三部分：与AI工具的协同模式

7. **如何高效地向AI提问？**
   - **好问题 vs 坏问题**：
     - ❌ 坏："帮我做个登录功能"（太笼统）
     - ✅ 好："我需要在frontend/src/app/login/page.tsx中实现登录功能，要求：1）使用邮箱+密码登录，2）调用后端/api/auth/login接口，3）登录成功后跳转到/workspace，4）使用JWT存储在localStorage"
   - **提问模板**：
     - 背景：当前项目情况
     - 需求：具体要做什么
     - 约束：技术栈、已有代码、不能改的部分
     - 期望：预期结果
   - **如何提供上下文**：
     - 粘贴相关文件内容
     - 说明文件位置
     - 描述现有功能
     - 提供报错信息

8. **如何让AI生成高质量代码？**
   - **明确技术栈和规范**：
     - "使用Next.js 14 App Router，不要用Pages Router"
     - "使用TypeScript，不要用JavaScript"
     - "使用Tailwind CSS，不要用内联样式"
     - "使用Knex.js操作数据库，不要用原生SQL"
   - **要求AI解释代码**：
     - "生成代码后，请逐行解释每段代码的作用"
     - "标注出关键逻辑和注意事项"
   - **要求AI考虑边界情况**：
     - "考虑用户输入为空的情况"
     - "考虑网络请求失败的情况"
     - "考虑并发访问的情况"

9. **如何审查AI生成的代码？**
   - **基础检查清单**：
     - [ ] 代码能否运行？（有无语法错误）
     - [ ] 是否符合项目规范？（命名、格式、结构）
     - [ ] 是否有安全问题？（SQL注入、XSS、CSRF）
     - [ ] 是否有性能问题？（N+1查询、内存泄漏）
     - [ ] 是否有边界情况处理？（空值、错误、异常）
   - **如何让AI帮你审查**：
     - "请审查这段代码，指出潜在问题"
     - "这段代码是否符合最佳实践？"
     - "这段代码有什么安全风险？"
   - **如何迭代优化**：
     - "这段代码性能不好，如何优化？"
     - "这段代码太复杂，如何简化？"
     - "这段代码不符合DRY原则，如何重构？"

### 第四部分：App开发特殊挑战

10. **如何选择App开发框架？**
    - **React Native**：
      - 优势：基于React，与现有Web项目技术栈一致，代码复用
      - 劣势：性能略低于原生，第三方库质量参差不齐
      - 适用场景：快速开发、预算有限、团队熟悉React
    - **Flutter**：
      - 优势：性能好、UI精美、跨平台体验一致
      - 劣势：Dart语言学习成本、与现有项目技术栈不一致
      - 适用场景：追求性能和UI体验
    - **原生开发（Swift + Kotlin）**：
      - 优势：性能最好、体验最佳、功能最全
      - 劣势：开发成本高、需要维护两套代码
      - 适用场景：大厂、预算充足、追求极致体验
    - **混合方案（WebView + Native壳）**：
      - 优势：开发成本低、复用Web代码
      - 劣势：性能差、体验差
      - 适用场景：不推荐（用户体验太差）
    - **推荐方案**：React Native（与现有技术栈一致，开发效率高）

11. **如何让AI帮你开发App？**
    - **React Native特殊问题**：
      - 如何处理导航（React Navigation）
      - 如何处理状态管理（Redux Toolkit / Zustand）
      - 如何处理原生功能（Camera、相册、文件系统）
      - 如何处理推送通知
      - 如何处理支付（微信支付、支付宝）
    - **AI工具选择**：
      - 架构设计：GPT-5 Pro
      - 代码生成：Claude Code（React Native支持好）
      - 日常开发：Cursor
      - UI生成：v0.dev（生成React组件后改为React Native）
    - **开发流程**：
      - 第1步：用GPT-5 Pro设计App架构（页面结构、导航、状态管理）
      - 第2步：用Claude Code生成核心页面和逻辑
      - 第3步：用Cursor日常迭代和调试
      - 第4步：用真机测试（iOS模拟器 + Android模拟器）

12. **如何复用Web代码到App？**
    - **可复用的部分**：
      - 业务逻辑（API调用、数据处理）
      - 状态管理（Redux store）
      - 工具函数（utils）
      - 类型定义（TypeScript types）
    - **不可复用的部分**：
      - UI组件（Web用div/button，App用View/TouchableOpacity）
      - 样式（Web用CSS，App用StyleSheet）
      - 路由（Web用Next.js Router，App用React Navigation）
    - **如何设计复用架构**：
      - Monorepo结构（packages/shared、packages/web、packages/app）
      - 共享业务逻辑和API层
      - 各自实现UI层

### 第五部分：常见陷阱与避坑指南

13. **0基础开发者最常犯的错误**
    - **错误1：盲目信任AI生成的代码**
      - 问题：AI生成的代码可能有bug、安全问题、性能问题
      - 解决：让AI解释代码、让AI审查代码、自己测试验证
    - **错误2：不理解就复制粘贴**
      - 问题：代码跑不通时不知道如何调试
      - 解决：让AI逐行解释，确保理解后再使用
    - **错误3：不使用版本控制（Git）**
      - 问题：改坏代码后无法回退
      - 解决：每次修改前git commit，随时可以回退
    - **错误4：不写注释**
      - 问题：过几天就忘了代码是干什么的
      - 解决：让AI生成注释，关键逻辑必须注释
    - **错误5：不测试就部署**
      - 问题：线上出bug影响用户
      - 解决：本地测试 → 测试环境 → 生产环境

14. **如何避免把项目搞崩溃？**
    - **Git保护策略**：
      - 主分支（main/master）受保护，不直接修改
      - 新功能在feature分支开发
      - 测试通过后再合并到主分支
    - **备份策略**：
      - 每天git push到远程仓库
      - 重要节点打tag（如：v1.0.0）
      - 数据库定期备份
    - **测试策略**：
      - 本地测试：npm run dev测试功能
      - 单元测试：关键逻辑写测试（让AI帮你写）
      - 集成测试：测试环境跑一遍完整流程

15. **如何持续学习和进步？**

      - 遇到问题就问AI（不要自己瞎摸索）

### 第六部分：实战案例与最佳实践

16. **案例1：0基础开发者如何添加一个新页面？**
    - **需求**：添加"我的订单"页面
    - **步骤**：
      1. 向Claude Code描述需求："我需要在frontend/src/app下添加一个orders页面，显示当前用户的所有订单，包括订单号、金额、状态、创建时间"
      2. Claude Code生成：
         - frontend/src/app/orders/page.tsx（页面组件）
         - frontend/src/app/orders/OrderList.tsx（订单列表组件）
         - backend/src/routes/orders.js（后端路由）
         - backend/src/services/order.service.js（业务逻辑）
      3. 本地测试：npm run dev，访问http://localhost:3000/orders
      4. 调试修改：如果有问题，向Claude Code描述问题，让它修复
      5. Git提交：git add . && git commit -m "feat: 添加我的订单页面"

17. **案例2：如何修复一个生产环境的bug？**
    - **问题**：用户反馈"提交任务后一直显示处理中，但实际已完成"
    - **步骤**：
      1. 复现问题：在本地尝试复现
      2. 向Claude Code描述问题："用户提交任务后，前端一直显示processing状态，但后端任务已完成。涉及文件：frontend/src/app/workspace/TaskStatus.tsx, backend/src/services/task.service.js"
      3. Claude Code分析：可能是WebSocket未正确推送状态更新
      4. Claude Code生成修复方案
      5. 本地测试：验证修复有效
      6. 部署到生产：git push后触发CI/CD

18. **案例3：如何设计并开发一个复杂功能（CMS系统）？**
    - **需求**：后台管理系统，管理功能卡片、公告、轮播图
    - **步骤**：
      1. 向GPT-5 Pro提问："请设计一个CMS系统架构，支持功能卡片管理、公告管理、轮播图管理。技术栈：Next.js + Express.js + MySQL"
      2. GPT-5 Pro生成架构设计（数据库设计、API设计、页面设计）
      3. 向Claude Code提供架构设计："根据以下架构设计，生成代码..."
      4. Claude Code分阶段生成代码：
         - 第1阶段：数据库表和Model
         - 第2阶段：后端API
         - 第3阶段：前端页面和组件
      5. 逐个功能测试、调试、优化
      6. 集成测试、上线
