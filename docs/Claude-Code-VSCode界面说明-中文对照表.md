## 🔥 三个模式详细说明

### 模式 1：⏵ Edit automatically（自动编辑模式）

**图标**：⏵⏵（双箭头）

**中文名**：自动编辑模式

**工作方式**：
- ✅ Claude **直接编辑文件**，不问你
- ✅ Claude **直接运行命令**，不问你
- ✅ 速度最快，无打断

**适用场景**：
- ✅ 你完全信任 Claude
- ✅ 简单任务（如格式化代码）
- ✅ 需要快速迭代

**风险**：
- ⚠️ Claude 可能改错代码
- ⚠️ 可能运行危险命令
- ⚠️ 需要你事后检查

**启用方式**：
```
按 Shift + Tab → 看到 ⏵⏵ accept edits on（自动接受编辑已开启）
```

**关闭方式**：
```
再按 Shift + Tab → 切换到其他模式
```

---

### 模式 2：📋 Plan mode（计划模式）

**图标**：⏸（暂停图标）

**中文名**：计划模式

**工作方式**：
1. Claude 先**分析任务**
2. Claude **制定详细计划**
3. **显示计划给你看**
4. **问你**：是否执行这个计划？
5. 你确认后，Claude 才开始执行

**适用场景**：
- ✅ 复杂任务（如重构代码）
- ✅ 新项目开发
- ✅ 你想先看计划再决定

**优势**：
- ✅ 计划清晰，心里有数
- ✅ 可以修改计划
- ✅ 避免 Claude 乱做

**工作流程示例**：
```
你："请重构这个函数"

Claude（计划模式）：
  📋 计划：
  1. 分析当前函数逻辑
  2. 提取重复代码
  3. 创建辅助函数
  4. 编写单元测试

  是否执行此计划？[Y/n]

你："Y"（确认）

Claude：[开始执行计划]
```

**启用方式**：
```
按 Shift + Tab → 看到 ⏸ plan mode on（计划模式已开启）
```

---

### 模式 3：普通模式（Normal mode）

**图标**：无特殊图标

**中文名**：普通模式 / 默认模式

**工作方式**：
- Claude **每次操作都问你**
- 编辑文件前问你：是否允许？
- 运行命令前问你：是否运行？
- 你确认后才执行

**适用场景**：
- ✅ 默认模式（最安全）
- ✅ 学习阶段（了解 Claude 做什么）
- ✅ 敏感操作（如删除文件）

**优势**：
- ✅ 完全可控
- ✅ 最安全
- ✅ 适合新手

**缺点**：
- ⚠️ 需要频繁确认
- ⚠️ 速度较慢

**示例**：
```
你："请删除这个文件"

Claude：
  ⚠️ 我即将删除文件：src/old-code.js
  是否继续？[Y/n]

你："Y"（确认）

Claude：[删除文件]
```

**启用方式**：
```
按 Shift + Tab → 切换回普通模式（没有特殊图标显示）
```

## ⚠️ 重要提醒

### 安全建议：

1. **新手**：始终使用**普通模式**
2. **敏感操作**：切换到**普通模式**（删除文件、修改配置）
3. **快速迭代**：使用**自动编辑模式**（格式化、简单修改）
4. **复杂任务**：使用**计划模式**（重构、新功能）

### 风险警告：

⚠️ **自动编辑模式**下：
- Claude 可以删除任何文件
- Claude 可以运行任何命令
- Claude 可以修改系统配置

**老王建议**：
- ✅ 先用**计划模式**看看 Claude 要做什么
- ✅ 确认安全后，再切换到**自动编辑模式**
- ✅ 重要操作前先**备份代码**（Git commit）
